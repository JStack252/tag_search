# Contributing to Tag Search

Thank you for your interest in contributing to Tag Search. Contributions involving code, documentation, design, testing, and tag organization are welcome.

## Reporting Bugs

Before reporting a bug, please check the existing Issues to make sure it has not already been reported.

When creating a bug report, please include:

- A clear description of the problem
- Steps to reproduce the problem
- The expected result
- The actual result
- Screenshots or examples, if available

## Suggesting Features

Feature suggestions should be submitted as GitHub Issues.

Please explain:

- What feature you are suggesting
- What problem it would solve
- Who would benefit from it
- An example of how it could work

## Claiming an Issue

Before beginning work, comment on the open Issue to let other contributors know that you would like to work on it.

Please wait for a maintainer to confirm or assign the Issue before submitting major changes.

## Submitting a Pull Request

1. Fork the repository.
2. Create a new branch for your contribution.
3. Make and review your changes.
4. Commit your changes with a clear commit message.
5. Open a Pull Request to the original repository.
6. Link the Pull Request to the related Issue, when applicable.

Please keep each Pull Request focused on one task.

## Respectful Collaboration

All contributors should communicate respectfully and provide constructive feedback.

Harassment, discrimination, personal attacks, and disrespectful behavior will not be accepted.

## Documentation and Coding Style

- Use clear and understandable English.
- Keep documentation concise and beginner-friendly.
- Use descriptive file, variable, and function names.
- Add comments when code may be difficult to understand.
- Follow the existing project structure and style.
